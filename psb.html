Módulo 01

Teoria e Conceitos

Os conceitos abaixo formam a base para toda a disciplina. Clique em cada tópico para expandir o conteúdo.

O ATMEGA328P é um microcontrolador (µC) RISC de 8 bits baseado na arquitetura AVR RISC, que por sua vez é uma implementação da arquitetura de Harvard, que separa a memória de Programa da memória de Dados. Por se tratar de um µC de arquitetura RISC, uma de suas principais caracteristicas é que apenas as instruções LOAD e STORE tem a capacidade de acessar registradores da Memória de dados.

Especificações Principais

  • Arquitetura: AVR RISC de 8 bits
  • Instruções: 131
  • Memória de Programa: 32 KB
  • Memória de Dados: 2304 B
  • SRAM (RAM): 2 KB
  • I/O Externo: 160 B
  • I/O Interno: 64 B
  • Registradores de propósito geral: 32
  • EEPROM (ROM): 1 KB
  • Pinos de I/O: 23

Os registradores de propósito geral são nomeados de R0 à R31. Os seis últimos podem ser combinados para formar registradores de 16 bits: X [R27:R26], Y [R29:R28] e Z [R31:R30]. O registrador par armazena os 8 bits menos significativos (LSB) e o ímpar os 8 bits mais significativos (MSB).

Registradores mais importantes do I/O interno

SREG (Status Register)

O SREG reporta, em cada um de seus bits, como foi a conclusão da última instrução executada:

  • C: ocorrência de overflow em operações sem sinal (unsigned).
  • Z: resultado da operação é zero.
  • N: resultado negativo.
  • V: overflow em operações com sinal (signed).
  • S: sinal lógico do resultado (0 = positivo, 1 = negativo).
  • H: overflow do primeiro para o segundo nibble.
  • T: bit temporário para cópia.
  • I: controle global de interrupções.

SP (Stack Pointer)

O SP armazena o endereço da SRAM correspondente ao topo da pilha. No AVR, utiliza o modelo Full Descending Stack, onde o SP aponta para o último endereço ocupado.

  • PUSH: pré-decrementa o SP e armazena o dado.
  • POP: carrega o dado e pós-incrementa o SP.

O SP é implementado como dois registradores de I/O: SPH e SPL, totalizando 12 bits de endereçamento.

Assim como na arquitetura x86, as instruções de controle de fluxo podem ser divididas em desvios incondicionais e condicionais.

Desvio Incondicional

Executam o salto independentemente do estado do processador.

JMP (Jump) — salto para endereço absoluto
  • Tamanho: 2 palavras (4 bytes)
  • Clock: 3 ciclos
  • Uso: endereça toda memória de programa
RJMP (Relative Jump) — salto relativo (PC ← PC + k + 1)
  • Tamanho: 1 palavra (2 bytes)
  • Clock: 2 ciclos
  • Uso: loops curtos e desvios locais
IJMP (Indirect Jump) — salto via registrador Z
  • Tamanho: 1 palavra (2 bytes)
  • Clock: 2 ciclos
  • Uso: jump tables e ponteiros de função

Desvio Condicional

O salto depende de uma comparação prévia e das flags do SREG.

1. Instruções de Comparação

CPI — compara registrador com constante (Rd − K)
CPC — compara registradores com carry (Rd − Rr − C)
CPSE — pula próxima instrução se iguais

2. Instruções de Branch (BRxx)

BRxx — salto relativo baseado no SREG
  • Tamanho: 1 palavra
  • Clock: 1 ciclo (falso) / 2 ciclos (verdadeiro)
  • Uso: estruturas como if, while, for
Igualdade
BREQ (igual) | BRNE (diferente)
Sem sinal
BRLO (menor) | BRSH (maior ou igual)
Com sinal
BRLT (menor) | BRGE (maior ou igual)
Overflow
BRVS (com overflow) | BRVC (sem overflow)

Laboratório Virtual

SimulIDE:
Prática e Validação

O SimulIDE é um simulador de circuitos eletrônicos em tempo real que permite visualizar a execução de códigos Assembly em hardware simulado. É a ferramenta central dos laboratórios da disciplina.

  • check_circle Simulação lógica de alta performance
  • check_circle Debug visual de registradores e memória
  • check_circle Suporte a múltiplos microcontroladores
  • check_circle Integração com código Assembly NASM
Como usar no SimulIDE
1 Crie seu arquivo .asm com NASM e compile com nasm -f elf64
2 Importe o binário no SimulIDE pelo menu File → Load Firmware
3 Use os breakpoints visuais para inspecionar registradores ciclo a ciclo
download Baixar SimulIDE v0.4.15

Configuração do Ambiente

Instruções de Base

Guia passo a passo para configurar seu ambiente de desenvolvimento

01

Toolchain

Instalação do NASM e GCC para compilação de código Assembly e integração com C.

# Ubuntu/Debian
$ sudo apt install nasm gcc
# verificar
$ nasm --version
02

Ambiente

Configuração do VS Code com extensões de syntax highlighting e debuggers específicos.

ASM Code Lens
x86 and x86_64 NASM
Native Debug (GDB)
03

Compilação

Inclusão das headers padrão e procedimento de montagem e linkedição do binário final.

# montar
$ nasm -f elf64 prog.asm
# linkar
$ ld -o prog prog.o
# executar
$ ./prog
04

Depuração

Uso do GDB para depuração passo a passo, inspecionando registradores e memória em tempo real.

# compilar com debug
$ nasm -f elf64 -g prog.asm
$ gdb ./prog
(gdb) break _start
(gdb) run
(gdb) info registers
